LJ::load_userids_multiple

LJ::load_userids_multiple — Loads a number of users at once, efficiently.

Use

LJ::load_userids_multiple(dbarg, map, have, memcache_only);

Arguments

  • dbarg. Deprecated; a master database handle ($dbh) or a master/slave set of database handles ($dbs).

  • map. Arrayref of pairs (userid, destination scalarref).

  • have. Arrayref of user objects caller already has.

  • memcache_only. Flag to only retrieve data from memcache.

Info

loads a few users at once, their userids given in the keys of $map listref (not hashref: can't have dups). values of $map listref are scalar refs to put result in. $have is an optional listref of user object caller already has, but is too lazy to sort by themselves. Note: The $have parameter is deprecated, as is $memcache_only; but it is still preserved for now. Really, this whole API (i.e. LJ::load_userids_multiple) is clumsy. Use [func[LJ::load_userids] instead.

Source:

cgi-bin/LJ/User.pm

Returns:

Nothing.
