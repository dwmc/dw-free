LJ::make_postto_select

LJ::make_postto_select — Given a u object and some options, determines which users the given user can post to. If the list exists, returns a select list and a submit button with labels. Otherwise returns a hidden element.

Use

LJ::make_postto_select(u, opts);

Arguments

  • u. A hashref of a row from the user table, sometimes called a "user object". Although only initially a row from the user table, gets a lot of tacked onto it from various other LJ API functions.

  • opts. Optional. Valid keys are: 'postto' - current user, gets selected in drop-down; 'label' - label to go before form elements; 'button' - button label for submit button; others - arguments to pass to LJ::get_postto_list.

Source:

cgi-bin/weblib.pl

Returns:

string of HTML elements
